# 每日任務打勾 App ・棒球主題（設計文件）

日期：2026-06-22

## 一、目標與背景

為國小四年級的兒子做一個「每天勾選已完成任務」的工具，在 iPad 上使用。
兒子熱愛棒球，整體版面與互動採用棒球場主題。

**核心需求**
- 可編輯每天要做的任務。
- 任務以「每週重複」為基礎（每個星期幾各有一組固定範本）。
- 切換到的每一天都能單獨增刪修改，不影響每週範本，也不影響其他日子。
- 小孩完成任務後可以打勾。

**硬性限制**
- 在 iPad 上運行。
- 不需付費、不上架到 App Store。
- 資料只存在兒子的那一台 iPad 本機（不需跨裝置同步）。

## 二、技術方案

採用 **PWA（漸進式網頁應用）**：

- 純前端網頁（HTML / CSS / JavaScript），無後端、無建置工具相依。
- 用 Safari 開啟後「加入主畫面」，桌面會出現 App 圖示，點開為全螢幕（無網址列），體驗接近原生 App。
- 透過 Web App Manifest 設定圖示、名稱、全螢幕顯示模式。
- 透過 Service Worker 快取靜態檔案，達成**離線可用**。
- 資料儲存於瀏覽器 **localStorage**，存在 iPad 本機。
  - 與 iCloud 無關、不佔 iCloud 空間。
  - 任務資料為純文字，體積極小（數 KB 等級）。

**為何不選其他方案**
- 原生 App：需 App Store 上架或側載，不符限制。
- 雲端託管 / 跨裝置同步：需網路與帳號設定，使用者選擇不需要。

## 三、畫面結構

三個畫面：

### 1. 主畫面（小孩主要使用）
- **頂部日期列**：顯示日期與星期幾；「◀ 前一天 / 後一天 ▶」箭頭切換日期；「回到今天」按鈕。
- **計分板區**：棒球記分板樣式，顯示「完成 N / 總數」；一個小型內野鑽石（diamond），每完成一項任務跑者推進一個壘包。
- **任務清單**：每項任務一張圓角卡片，左側為「棒球」勾選元件。點一下打勾 → 棒球亮起（白底紅縫線）、蓋上紅色「SAFE / 安打」印記、卡片變綠。
- **底部按鈕**：
  - 「編輯這一天」：編輯**目前切換到的那一天**。
  - 「每週範本」：進入每週範本設定。
  - 按鈕以純文字搭配棒球風格樣式呈現，不使用 emoji。
- **全部完成**：當天任務全部打勾時，跳出「全壘打！今天全部完成！」橫幅與簡單動畫（棒球飛出）。

### 2. 編輯這一天
- 對象為**目前主畫面顯示的那一天**。
- 功能：新增任務、刪除任務、修改任務文字、調整順序。
- 只影響這一天，不動到每週範本，也不影響其他日子。

### 3. 每週範本設定
- 上方選擇星期一～星期日。
- 下方編輯該星期幾的固定任務清單（新增 / 刪除 / 修改 / 排序）。
- 影響「之後尚未被單獨修改過」的日子。

## 四、資料模型與運作邏輯

### 資料結構（localStorage，JSON）
- `templates`：以星期幾為鍵（0=星期日 … 6=星期六）的物件，每個值是任務陣列。
  - 任務：`{ id: string, title: string }`
- `days`：以日期字串 `YYYY-MM-DD` 為鍵的物件，每個值是當天任務陣列。
  - 任務：`{ id: string, title: string, done: boolean }`
  - 只有「被動過」（打勾或編輯）的日子才會存在於 `days`。

### 取得某一天任務的邏輯
1. 若 `days[date]` 存在 → 直接使用該天已儲存的清單。
2. 若不存在 → 依該日期的星期幾，從 `templates` 帶出範本任務，全部 `done = false`（此時為「虛擬」顯示，尚未寫入 `days`）。

### 寫入時機（虛擬 → 實體化）
- 在某天**打勾**或**編輯**時，將該天清單（含目前 done 狀態）寫入 `days[date]`。
- 自此該天獨立保存，之後修改範本不影響它。

### 行為規則
- 打勾狀態以「每一天」各自獨立，不跨日共用。
- 編輯某一天不影響其他日子與範本。
- 編輯範本只影響「之後尚未被動過」的日子；已實體化的日子不變。

## 五、外觀風格（棒球主題，無 emoji）

- **配色**：草皮綠、紅土褐、白色壘線、深藍計分板。
- **棒球勾選鈕**：未完成為灰色棒球；完成為白底紅縫線棒球並蓋上紅色「SAFE / 安打」印記，卡片底色轉綠。
- **進度**：頂部記分板「完成 N / 總數」；內野鑽石跑者隨完成數推進壘包。
- **完成動畫**：全部完成顯示「全壘打！今天全部完成！」橫幅與棒球飛出動畫。
- **字體**：圓潤、運動感的大字體；大按鈕、大點擊區，適合兒童觸控。
- **語言**：全繁體中文。
- 帶入棒球用語（上壘、安打、全壘打），但不影響易懂性。
- 全程不使用 emoji。

## 六、檔案結構（預計）

```
index.html          主頁面（三畫面以單頁切換）
styles.css          棒球主題樣式
app.js              邏輯：資料模型、畫面切換、打勾、編輯、範本
manifest.webmanifest  PWA 設定（名稱、圖示、全螢幕）
service-worker.js   離線快取
icons/              App 圖示（棒球主題）
```

## 七、測試與驗證

- 在桌機瀏覽器以行動裝置模擬尺寸驗證互動。
- 重點驗證：
  - 切換日期能正確帶出對應星期幾範本。
  - 打勾後重整頁面仍保留（localStorage 持久化）。
  - 編輯某天不影響其他天與範本。
  - 編輯範本只影響之後未動過的日子。
  - 全部完成觸發全壘打動畫。
  - 加入主畫面後可離線開啟使用。
- 最終於 iPad Safari 實機驗證「加入主畫面 → 離線使用 → 打勾持久化」。

## 八、不做的範圍（YAGNI）

- 不做跨裝置同步、雲端、帳號登入。
- 不做密碼/PIN 保護編輯。
- 不做歷史統計報表（僅可切換日期查看）。
- 不做提醒通知。
